01 สองเส้นทาง deploy

ตัวแปรตัดสินคือ นโยบายข้อมูลของมหาวิทยาลัย — จะฝากบน cloud ได้ หรือต้องเก็บในองค์กร ทั้งสองเส้นทางใช้ PostgreSQL เหมือนกัน ฉะนั้น data model ออกแบบครั้งเดียว ย้ายภายหลังได้โดยไม่รื้อ

Cloud Managed

แนะนำเริ่มเร็ว

Supabase — เหมาะกับทีมเล็ก อยากเห็นของใช้จริงเร็ว ไม่ติดข้อบังคับ on-prem

  • ฐานข้อมูลPostgreSQL (จัดการให้)
  • AuthSupabase Auth (พร้อมใช้)
  • ไฟล์Supabase Storage
  • สิทธิ์Row-Level Security map จาก ROLES ตรงๆ
  • งานหลังบ้านน้อยมาก — auto REST / Realtime

On-prem

ถ้าต้องเก็บในองค์กร

เซิร์ฟเวอร์มหาวิทยาลัย — เหมาะเมื่อ IT บังคับให้ข้อมูลอยู่ในองค์กร

  • ฐานข้อมูลPostgreSQL (self-host)
  • AuthJWT — NestJS / FastAPI
  • ไฟล์MinIO (S3-compatible)
  • สิทธิ์guard ในชั้น API
  • งานหลังบ้านมากกว่า — ต้องเขียน API เอง
➡

ระบบสิทธิ์ที่ออกแบบไว้แล้ว (admin/user + roles รายโครงการ + capability union) map เข้ากับ Supabase Row-Level Security ได้พอดี — ประหยัดงานหลังบ้านมาก จึงเชียร์ Supabase เป็นค่าเริ่มต้น ยกเว้นมหาวิทยาลัยบังคับ on-prem

02 ทำไม PostgreSQL

ข้อมูลของระบบนี้เป็น relational ชัดเจน (โครงการ → กลุ่ม → กรง → หนู → น้ำหนัก/การรักษา/ชันสูตร) และต้องคำนวณค่าเฉลี่ยกลุ่ม เทียบ control ตลอด — งานแบบ join + aggregate นี้ NoSQL/Firestore ทำได้ลำบาก Postgres จึงเป็นแกนที่เหมาะที่สุด

03 การแนบเอกสาร PDF

แต่ละโครงการแนบเอกสารสำคัญได้ (โปรโตคอล · ใบอนุมัติ EC · SOP · ผลแล็บ) — เดโมหน้าตาพร้อมแล้วในตัว prototype ส่วนของจริงใช้แนวทางมาตรฐานคือแยกไฟล์ออกจากฐานข้อมูล

● ในเดโมตอนนี้ (ทำแล้ว)

  • ปุ่ม เอกสาร ในหน้าโครงการ — แนบ / เปิด / ลบ PDF
  • เก็บไฟล์เป็น object URL ในหน่วยความจำ
  • คุมสิทธิ์: ดูได้ทุกคนที่เข้าถึง · แนบ/ลบเฉพาะ PI
  • ข้อจำกัด: รีเฟรชแล้วหาย — ไว้โชว์ UX เท่านั้น

● ในระบบจริง

  • ตัวไฟล์ → object storage (ไม่เก็บ BLOB ในฐานข้อมูล)
  • metadata → ตาราง documents
  • เปิดผ่าน signed URL (หมดอายุได้) คุมสิทธิ์รายโครงการ
  • แนบได้ทั้งระดับโครงการและระดับเคส (สแกนใบชันสูตร ฯลฯ)
-- ตารางเก็บ metadata ของไฟล์ (ตัวไฟล์อยู่บน object storage) create table documents ( id uuid primary key, project_id uuid references projects, mouse_id uuid references mice, -- null = เอกสารระดับโครงการ filename text, mime text, size bigint, storage_key text, -- path บน storage category text, -- protocol · ec · sop · lab uploaded_by uuid, uploaded_at timestamptz );
